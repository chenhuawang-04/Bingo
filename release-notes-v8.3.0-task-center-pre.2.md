# Bingo v8.3.0-task-center-pre.2

## 全项目任务与资源审计

- 所有 OkHttp 客户端改为共享进程级 Dispatcher，并自动登记网络资源租约。
- 内存、CPU 和数据库写入增加进程级挂起式准入门，前台重型操作之间不再无约束争抢。
- OED WebView WAF 解题、图片压缩、PDF 渲染、导入导出和 TTS 预热纳入统一资源快照。
- 后台任务管理页新增实时内存、CPU、网络、数据库写入和音频占用展示。
- TTS AudioFocus 生命周期接入音频资源租约，独占型重任务会避让正在进行的朗读。

## 性能与稳定性

- TTS 长文章预热改为固定工作协程池，取消旧会话，避免为每个段落创建协程。
- TTS 引擎等待和播放等待增加超时；回调表在取消时必定清理。
- MediaPlayer 使用异步准备，消除主线程同步磁盘读取风险。
- TTS 缓存限制为 200 MiB，并按最旧文件自动淘汰。
- PDF 最多处理 40 页，限制 DPI、长边和像素数；图片批次限制数量、单文件大小和总结果大小。
- 导入导出文件读写移至 IO 调度器，导入大小受限，并登记数据库写入资源。
- 修复后台整理任务 ID 映射的跨线程数据竞争。
- UI 协程不再吞掉生命周期取消异常。

## 兼容性

- Room schema 保持版本 36，无数据库迁移。
- `versionCode` 更新为 14，`versionName` 更新为 `8.3.0-task-center-pre.2`。
