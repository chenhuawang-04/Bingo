# Bingo Development Board

Kanban board for tracking feature development, bug fixes, and improvements for the Bingo English vocabulary learning app.

## Columns

| Column | Description |
|--------|-------------|
| **Backlog** | Ideas and proposals not yet scheduled |
| **Todo** | Accepted tasks ready for the next development cycle |
| **In Progress** | Currently being worked on |
| **In Review** | Code complete, pending review or testing |
| **Done** | Merged and shipped |

## Labels

| Label | Meaning |
|-------|---------|
| `feature` | New functionality |
| `enhancement` | Improvement to existing feature |
| `bug` | Something isn't working |
| `refactor` | Code quality / architecture improvement |
| `test` | Test coverage addition |
| `docs` | Documentation update |
| `P1` | Must have — blocks release |
| `P2` | Should have — important but not blocking |
| `P3` | Nice to have — optional |

## Milestones

- **v1.1** ✅ — Article module, vocabulary linking, pagination, code quality improvements
- **v1.2** — _Planned next release_

## How to Use

1. New ideas go to **Backlog** as Issues with appropriate labels
2. During planning, move prioritized items to **Todo** and assign a Milestone
3. When you start working, move to **In Progress** and assign yourself
4. After completing, move to **In Review** for testing
5. Once verified, move to **Done**
